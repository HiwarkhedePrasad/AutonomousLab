Foundation

What is MCP?

Model Context Protocol — the open standard that lets AI assistants talk to your systems.

PROTOCOL
JSON-RPC 2.0 Core

MCP uses JSON-RPC 2.0 as its communication backbone. Every request and response follows a strict schema: jsonrpc, method, params, and id.

→
TOOLS
Custom Functions

You define tools with names, descriptions, and JSON Schema input definitions. ChatGPT discovers and calls them automatically via tools/list and tools/call.

→
AUTH
OAuth 2.0 + PKCE

Production servers use OAuth 2.0 Authorization Code Flow with PKCE. ChatGPT acts as the OAuth client, your server is the provider.

→
STATE
Persistent Sessions

Maintain user context across conversations. Tokens, refresh cycles, and session management give ChatGPT memory of who the user is.

02
ChatGPT Configuration

Enable Developer Mode

Six precise steps to connect ChatGPT to your MCP server.

1

Open ChatGPT Settings

Navigate to https://chatgpt.com. Click your profile icon in the top-right corner, then select Settings from the dropdown menu.

2

Navigate to Connectors

In Settings, click Connectors → Advanced Settings. Toggle on Developer Mode (beta). This unlocks the connector creation UI.

3

Create a New Connector

Click Create next to "Browser connectors". A modal appears. Fill in: an optional icon (128×128px), a Name, and an optional description.

4

Enter Your MCP Server URL

In MCP Server URL, enter your deployed server: e.g. https://your-app.fly.dev. Set Authentication to OAuth. Check "I trust this application".

5

Complete OAuth Authorization

Click Create. You'll be redirected to your server's login page. Sign in or use the demo account. Grant permissions. You'll be redirected back to ChatGPT automatically.

6

Activate in a Chat

Start a new chat. In the model selector click ... More › → Developer Mode → select your connector name. All your tools are now available.

03
Critical Constraints

Requirements & Limitations

Miss any of these and your server will silently fail. Read carefully.

⚠ Critical Requirements — All Must Be Met
  • Required Tools: search and fetch MUST exist in tools/list or ChatGPT shows "search action not found"
  • OAuth Discovery: Must be at /.well-known/oauth-authorization-server on the root domain
  • MCP Endpoint: POST handler MUST be at root path / — not /mcp, not /api
  • UUID Client IDs: ChatGPT sends UUID-format client IDs — your server must accept any valid UUID
Platform Root Domain Control OAuth Support Verdict
Fly.io ✓ Full control ✓ Works perfectly Recommended
Railway / Render ✓ Full control ✓ Works well Good choice
Vercel Functions ⚠ Requires config ⚠ Can work Use with care
Supabase Edge Functions ✗ Served at /functions/v1/ ✗ Breaks OAuth discovery Do not use

Server Code

Implementation

Two complete options: simple (no auth) and production (OAuth 2.0).

server.js
import express from 'express';
import cors from 'cors';

const app = express();
const PORT = process.env.PORT || 8080;
const PROTOCOL_VERSION = "2025-06-18";

app.use(cors({
  origin: ['https://chatgpt.com', 'https://chat.openai.com'],
  credentials: true,
  allowedHeaders: ['Content-Type', 'Authorization', 'MCP-Protocol-Version'],
  methods: ['GET', 'POST', 'OPTIONS']
}));

app.use(express.json());

// MCP Protocol Handler — MUST be at root path /
app.post('/', (req, res) => {
  const { jsonrpc, method, params = {}, id } = req.body;

  switch (method) {
    case 'initialize':
      res.json({
        jsonrpc: '2.0', id,
        result: {
          protocolVersion: PROTOCOL_VERSION,
          capabilities: { tools: { listChanged: false } },
          serverInfo: { name: 'mcp-demo', version: '1.0.0' }
        }
      });
      break;

    case 'tools/list':
      res.json({
        jsonrpc: '2.0', id,
        result: {
          tools: [
            // REQUIRED: search and fetch MUST be present
            {
              name: 'search',
              title: 'Search',
              description: 'Search for information',
              inputSchema: {
                type: 'object',
                properties: {
                  query: { type: 'string', description: 'Search query' }
                },
                required: ['query'],
                additionalProperties: false
              }
            },
            {
              name: 'fetch',
              title: 'Fetch Data',
              description: 'Fetch data by ID',
              inputSchema: {
                type: 'object',
                properties: {
                  id: { type: 'string', description: 'Resource ID' }
                },
                required: ['id'],
                additionalProperties: false
              }
            }
          ]
        }
      });
      break;

    case 'tools/call':
      const { name, arguments: args = {} } = params;
      let content;

      if (name === 'search') {
        content = [{ type: 'text', text: JSON.stringify([
          { id: '1', title: 'Result 1', content: `Found: ${args.query}` }
        ])}];
      } else if (name === 'fetch') {
        content = [{ type: 'text', text: JSON.stringify({
          id: args.id, data: `Data for ${args.id}`,
          timestamp: new Date().toISOString()
        })}];
      }

      res.json({ jsonrpc: '2.0', id, result: { content, isError: false } });
      break;

    default:
      res.json({ jsonrpc: '2.0', id,
        error: { code: -32601, message: 'Method not found' }
      });
  }
});

app.listen(PORT, () => console.log(`🚀 MCP Server on port ${PORT}`));
server-oauth.js — Key OAuth Endpoints
// 1. OAuth Discovery — MUST be at root domain /.well-known/
app.get('/.well-known/oauth-authorization-server', (req, res) => {
  res.json({
    issuer: BASE_URL,
    authorization_endpoint: `${BASE_URL}/authorize`,
    token_endpoint: `${BASE_URL}/token`,
    registration_endpoint: `${BASE_URL}/register`,
    response_types_supported: ['code'],
    grant_types_supported: ['authorization_code', 'refresh_token'],
    code_challenge_methods_supported: ['S256', 'plain'],
    scopes_supported: ['mcp:read', 'mcp:write']
  });
});

// 2. Dynamic Client Registration
app.post('/register', (req, res) => {
  const clientId = randomUUID();
  const client = {
    client_id: clientId,
    redirect_uris: req.body.redirect_uris || [],
    grant_types: req.body.grant_types || ['authorization_code'],
    token_endpoint_auth_method: req.body.token_endpoint_auth_method || 'client_secret_basic'
  };
  clients.set(clientId, client);
  res.status(201).json({ client_id: clientId, ...client });
});

// 3. Auto-register ChatGPT's UUID client IDs
function isChatGPTClient(clientId) {
  return clientId === 'chatgpt' ||
    /^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i.test(clientId);
}

// 4. Token Endpoint
app.post('/token', (req, res) => {
  const { grant_type, code, client_id, redirect_uri } = req.body;

  if (grant_type === 'authorization_code') {
    const authCode = authCodes.get(code);
    if (!authCode || authCode.expires_at < Date.now())
      return res.status(400).json({ error: 'invalid_grant' });

    const accessToken = randomUUID();
    const refreshToken = randomUUID();
    tokens.set(accessToken, { client_id, user: authCode.user, expires_at: Date.now() + 3600000 });
    authCodes.delete(code);

    res.json({
      access_token: accessToken,
      token_type: 'Bearer',
      expires_in: 3600,
      refresh_token: refreshToken,
      scope: authCode.scope
    });
  }
});

// 5. Protect MCP routes with token validation
function validateToken(authHeader) {
  if (!authHeader?.startsWith('Bearer ')) return null;
  const token = authHeader.substring(7);
  const data = tokens.get(token);
  return (data && data.expires_at > Date.now()) ? data : null;
}

// Include ALL ChatGPT redirect URIs
redirect_uris: [
  'https://chatgpt.com/oauth/callback',
  'https://chat.openai.com/oauth/callback',
  'https://chatgpt.com/connector_platform_oauth_redirect'
]

Production

Deploy to Fly.io

Four commands to go from local to live with full HTTPS and root domain control.

1

Install Fly CLI & Login

Run curl -L https://fly.io/install.sh | sh to install. Then flyctl auth login (or flyctl auth signup for new accounts). This stores your credentials locally.

2

Create fly.toml

Place this in your project root. Key settings: internal_port = 8080, force_https = true, auto_stop_machines = false, and min_machines_running = 1.

3

Set Secrets & Launch

Run flyctl secrets set SUPABASE_URL="..." SUPABASE_ANON_KEY="..." to inject environment variables securely. Then flyctl launch --name your-mcp-server --region iad.

4

Deploy & Verify

Run flyctl deploy --ha=false to deploy a single machine (cheaper for dev). Verify with: curl https://your-app.fly.dev/.well-known/oauth-authorization-server — you should get the JSON metadata back.

fly.toml
app = 'mcp-oauth-server'
primary_region = 'iad'

[http_service]
  internal_port = 8080
  force_https = true
  auto_stop_machines = false
  min_machines_running = 1

[[vm]]
  memory = '1gb'
  cpu_kind = 'shared'
  cpus = 1

[env]
  PORT = "8080"
  NODE_ENV = "production"

Common Pitfalls

Gotchas & Solutions

Every error you will encounter and how to fix it instantly.

"search action not found" — ChatGPT rejects connection +

Cause: Your tools/list response doesn't include a tool named search. ChatGPT validates this on connection.

Fix: Add both search and fetch tools to your tools/list handler. The names must be exactly "search" and "fetch" — case-sensitive, no variations.

"Only ChatGPT client is supported" — OAuth registration fails +

Cause: Your server only accepts a hardcoded client_id of "chatgpt" but ChatGPT sends a UUID like 47bc35b3-0d0f-4cbf-905f-d68125105b24.

Fix: Use the regex check: /^[0-9a-f]{8}-[0-9a-f]{4}-...$/.test(clientId) to accept any valid UUID as a legitimate ChatGPT client and auto-register it.

OAuth discovery returns 404 — Supabase Edge Functions used +

Cause: Supabase Edge Functions are served under /functions/v1/, so your discovery URL becomes /functions/v1/.well-known/... — but ChatGPT looks at the root domain only.

Fix: Migrate to Fly.io, Railway, or Render. These platforms give you full control over the root path. Supabase Edge Functions simply cannot host an MCP OAuth server that ChatGPT will accept.

"Invalid redirect_uri" — Authorization fails after login +

Cause: ChatGPT's redirect URI isn't whitelisted on your server's client registration.

Fix: Add all three variants to your allowed list: https://chatgpt.com/oauth/callback, https://chat.openai.com/oauth/callback, and https://chatgpt.com/connector_platform_oauth_redirect. ChatGPT may use any of them.

CORS error — Browser blocks requests from ChatGPT +

Cause: Missing or incorrect CORS headers on your server responses.

Fix: Configure CORS with origin: ['https://chatgpt.com', 'https://chat.openai.com'], credentials: true, and include MCP-Protocol-Version in allowedHeaders. Apply to ALL routes including the /.well-known/ endpoint.

Error Message Root Cause Quick Fix
Method not found Unimplemented MCP method in switch Add case to switch + default handler
Authentication required Missing or expired Bearer token Re-authorize in ChatGPT settings
invalid_grant Auth code expired or already used Codes expire in 10 min — restart flow
500 Internal Server Error Unhandled exception in tool handler Wrap tools/call in try-catch
08
Production Hardening

Security Best Practices

Before going live, check every item on this list.

ENV VARS
No Hardcoded Secrets

All secrets go through process.env. Use flyctl secrets set to inject Supabase keys, JWT secrets, and database URLs. Never commit .env files.

INPUT
Validate & Sanitize

Every tools/call argument must be validated against its JSON Schema before use. Never use eval() — use a proper math library like mathjs for calculation tools.

RATE LIMIT
Throttle All Endpoints

Use express-rate-limit to limit auth endpoints (stricter) and MCP endpoints separately. The OAuth /authorize endpoint is especially vulnerable to brute force.

TOKENS
Short Token Lifetimes

Access tokens: 1 hour. Refresh tokens: 14 days. Auth codes: 10 minutes. Always delete auth codes after first use — they're single-use by the OAuth spec.

HTTPS
TLS Everywhere

Set force_https = true in fly.toml. ChatGPT will not connect to HTTP endpoints. All tokens transmitted over the wire must be encrypted in transit.

LOGGING
Audit All Auth Events

Log every token issue, auth code generation, and failed authentication attempt with timestamps and client IDs. Use structured JSON logs for easy querying in production.
